SentimentScope Engine

An end-to-end, fully containerized machine learning pipeline deployed on AWS that predicts user sentiment in real-time utilizing custom Natural Language Processing techniques.

View Live Demo View GitHub Repo

The Problem & Solution

The Challenge

Analyzing user feedback at scale is difficult without an automated system. Standard models often fail to capture the nuances of raw text data without proper cleaning, and deploying these models globally requires a robust, scalable infrastructure that doesn't break under load.

The Solution

I engineered a custom NLP pipeline to clean and vectorize text using TF-IDF. The processed data is fed into a highly optimized Logistic Regression model. To ensure seamless scalability and global access, the entire Flask API environment is Dockerized and hosted on an AWS EC2 instance, with historical data safely archived in an S3 bucket.

Technical Deep Dive

Raw text data is inherently messy. Before feeding data to the model, I utilized the spaCy library for robust text cleaning, handling tokenization, stop-word removal, and lemmatization. For feature extraction, I implemented a TF-IDF (Term Frequency-Inverse Document Frequency) vectorizer to transform the text into a meaningful numerical format that the regression model can process efficiently.

The core intelligence is powered by a Logistic Regression model, chosen for its speed and effectiveness in binary classification tasks. The model achieved a 91% accuracy rate on the evaluation metrics. To eliminate the "it works on my machine" problem, I wrapped the Python Flask API, the trained model file (.pkl), and all dependencies into a Docker container.

# Example Dockerfile snippet FROM python:3.9-slim WORKDIR /app COPY requirements.txt . RUN pip install -r requirements.txt COPY . . EXPOSE 5000 CMD ["python", "app.py"]

The Docker image is deployed on an AWS EC2 instance, serving as a reliable endpoint for client applications. To create a continuous feedback loop for future model retraining, every prediction payload, along with its calculated confidence score (e.g., Mood: POSITIVE, Confidence: 0.96), is automatically serialized and archived into an Amazon S3 Bucket via the Boto3 library.
