Overview

The Engineering Goal

Developers often face fragmented tooling when trying to integrate diverse computer vision tasks. My objective was to create a centralized, production-grade application leveraging the state-of-the-art YOLO26, providing seamless object detection, instance segmentation, and classification via optimized web and programmatic interfaces.

The Approach

I engineered a unified core inference engine supporting all three major computer vision tasks. I then built dynamic deployment layers: an intuitive, interactive Streamlit Web UI for instant visual analysis, and a high-throughput FastAPI REST backend for robust, automated integration. The entire platform is Dockerized and optimized for reliable, auto-scaling deployment on AWS.

Technical Deep Dive

Instead of building separate apps, I engineered a shared, class-based interface to centralize YOLO26 inference logic. This allows both the Web UI and the API to utilize identical processing pipelines for object detection, segmentation, and classification, ensuring consistency across both endpoints. The engine efficiently manages model states, result rendering (bounding boxes, pixel-perfect masks, top-k predictions), and flexible configuration parameters.

I structured two powerful interaction options. The Streamlit Web UI provides an intuitive drag-and-drop experience for visual exploration, while the FastAPI backend delivers production-grade REST endpoints. This gives teams absolute flexibility—integrating programmatically via robust API calls (with auto-generated Swagger UI) or instantly testing models visually through the web dashboard.

The platform was meticulously optimized for AWS App Runner. To avoid prolonged startup times during cold starts (especially critical for large ML models), I configured the Docker environment to pre-load Nano models into the image, slashing startup times to under 10 seconds. We implemented automated health check endpoints and utilized intelligent model caching (st.cache_resource) within Streamlit to maximize runtime efficiency and minimize memory overhead.
